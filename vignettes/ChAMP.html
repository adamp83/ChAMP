1 Introduction

The ChAMP package is a pipeline that integrates currently available 450k and EPIC analysis methods and also offers its own novel functionality. It utilises the data import from .idat file and normal beta matrix; Quality Control Plots; SWAN1 and Functional Normalization2 functions offered by the minfi3 package.

In addition, the ChAMP package includes the Peak Based Correction (PBC) method4 and the BMIQ Normalization5 is set as the default method.

A number of other pipelines and packages are available for 450k or EPIC array analysis including IMA6, minfi3, methylumi7, RnBeads8 and watermelon9. However, ChAMP provide a comprehensive and compelete analysis pipeline from origin data file to final GSEA result, which would make other researchers’ work much easyer. Also, the new version ChAMP provided Shiny-based WebBrower Interactive analysis function, which should benifit even more scientists.

There are also some other advantages for ChAMP. First of all ChAMP is both available for both 450k or EPIC array. Also this package contains multiple novel functions. The singular value decomposition (SVD) method10 allows an in-depth look at batch effects and for correction of multiple batch effect the ComBat method11 has been implemented. For the identification of differentially methylated regions (DMRs) ChAMP offers the new Probe Lasso method12. Also another two effective DMR detection function Bumphunter13 and DMRcate14 is also integrated and set default option.

For the purpose of dealing with cell heterogeneity problem, we included two functions RefFreeEWAS15 and RefbaseEWAS16 into ChAMP. Finally, ChAMP has an additional function to analyse 450k or EPIC for copy number alterations17.

Above are functions exist already in old version ChAMP, which all have been enhanceed and checked carefully. Also, some new functions are included as well:

For users who need to find methylation Blocks18. New version ChAMP included a function to detect methylation Blocks. Another important function is GSEA, which would take genes related into DMPs and DMRs and do GSEA analysis. To see what pathways are enriched by significant CpGs or DMRs. Also, new version ChAMP incoporated FEM package19, which would calculate modules in pathways show differentially methylation changes between two phenotypes.

Apart from that, new version ChAMP provide a Shiny and Plotly based GUI framework, to help scientists to view ChAMP result. This is a interactive WebBrower based framework, call for Graph System locally or remotely. For example X11 for most Linux System.

2 Installation

It is essential that you have R 3.3 or above version already installed on your computer. ChAMP is a pipeline that utilises many Bioconductor packages that are currently available from CRAN and Bioconductor. For all of the steps of the pipeline to work make sure that you have ungraded Bioconductor to newest version (3.3) .

After you have installed R and Bioconductor properly, you can start to install ChAMP, The most easy way to install it is typing following code into linux bash:

source("https://bioconductor.org/biocLite.R")
biocLite("ChAMP")

If you have any problem with traffic with Bioconductor, the other option is install ALL dependence packages, then install ChAMP. There are a lot if packaged relied by ChAMP. You can use following command in R to install most of them:

source("http://bioconductor.org/biocLite.R")
biocLite(c("minfi","ChAMPdata","Illumina450ProbeVariants.db","sva","IlluminaHumanMethylation450kmanifest","limma","RPMM","DNAcopy","preprocessCore","impute","marray","wateRmelon","plyr","GenomicRanges","RefFreeEWAS","qvalue","isva","doParallel","bumphunter","quadprog","shiny","shinythemes","plotly","RColorBrewer","DMRcate","dendextend","IlluminaHumanMethylationEPICmanifest","FEM","matrixStats"))

Note that ChAMP call for newest version FEM package FEM_3.1.0.tar.gz instead of version 2.8.0/. If you failed to install version 3.1.0, you may need to download FEM_3.1.0.tar.gz package and install it with following code:

R CMD INSTALL FEM_3.1.0.tar.gz

After you did all above work, you may use code below to install ChAMP:

R CMD INSTALL ChAMP_2.0.1.tar.gz

After installation, you may easily load ChAMP package: [r eval=TRUE] library("ChAMP")

3 Test Data

The package contains a test dataset of HumanMethylation450 data which can be used to test functions available in ChAMP. This can be loaded by pointing the directory to the testDataSet:

testDir=system.file("extdata",package="ChAMPdata")

This dataset contains only 8 Samples, we will use it to show ChAMP functions’ usage blow.

4 Full Pipeline

ChAMP Pipeline Figure above outlines the steps in the ChAMP pipeline. Each step can be run individually as a separate function. This allows integration with other analysis pipelines. It also enables the user to save the results of each step for future reference or further analysis. Alternatively the full pipeline can be run at once with one command:

champ.process(directory = testDir)

When running the full pipeline through the champ.process() function a number of parameters can be adjusted. These parameters are related to all functions involved in ChAMP package, but not all parameters are covered, otherwise champ.process() function would be be too large.

Note that champ.process() may not always successfully run, because any small mistake on data set in it may result to Error. So if you get any problem in champ.process() you may try separated steps below.
